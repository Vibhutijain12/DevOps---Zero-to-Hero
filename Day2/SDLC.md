#### Software Development Lifecycle
SDLC (Software Development Life Cycle) is a structured process for planning, building, testing, and maintaining software to ensure quality and alignment with business goals.


#### Key Phases of the SDLC: 

**Planning and Requirements Gathering -**  Defining the project scope and understanding customer needs, such as launching a new kids' clothing feature for an e-commerce platform  
**Defining and Designing -** Converting requirements into formal documentation like the Software Requirement Specification (SRS), followed by High-Level Design (HLD) and Low-Level Design (LLD).

**Building (Development) -** Developers write the application code and commit it to a shared repository like Git.

**Testing -** The Quality Engineering (QE) team validates the application in staging environments to ensure product quality.

**Deployment -** Pushing the validated software to the production environment for end-users.

* The role of DevOps - 
DevOps is a culture focused on automation to improve the efficiency of applications. The developer and testers handle the core creation of code and validate the code quality. A DevOps engineer is primarily responsible for optimizing the building, testing, and deployment phases. By automating these stages and eliminating the manual interventions. A DevOps engineer accelerates the process and ensures high consistency and speed. 

* Agile: 
Agile is a software development approach where the team builds the software in small parts, tests continuously, and improves quickly on the basis of feedback. 
Instead of building everything at once, work is done step by step.

#### Traditional method: 
Plan everything → Build everything → Test at the end 

#### Agile method:
Plan small part → Build → Test → Improve → Repeat
