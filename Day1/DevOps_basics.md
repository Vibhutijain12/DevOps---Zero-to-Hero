#### What is DevOps? - There are 100+ definitions available in the market for DevOps. 
DevOps is defined as a culture that improves an organisation’s ability to deliver applications efficiently. 
Example - suppose there is a company, example.com, that is taking 10 days to deliver their version from v1 to v2. Everything is done, development, testing, but holding the application for 10 days to deliver is not a good practice.

Ideally, it should take hours/1-2 days. Ability to deliver the application efficiently. 

Definition - DevOps is a process of improving the application delivery by ensuring there is proper automation, quality, monitoring, and continuous testing.

**1** Automation - Reducing manual efforts to speed up the processes. 
**2** Quality - Ensuring the product meets the high standards.
**3** Monitoring - Keeping track of system performance and health. 
**4** Continuous Testing - test the application to ensure that it is bug-free and working seamlessly. 
       
#### Why DevOps? - 
When there was no DevOps concept, companies used to deploy their application from a local server to production manually by taking help from different team members, like the system administrator, build and release engineer, and server administrator.

* How did it work - 
Once the code is ready, the developer shares the code in the centralized library, the system administrator creates a server for code testing, and the server administrator used to deploy the app on the server with the help of the build and release engineer. Then, the tester used to do their testing, and once everything looked good, they deployed the application to production. 

This approach took a lot of time to deliver the application to the customer, which is why DevOps came into existence. 

The evolution of DevOps breaks down silos between different roles like system administrators, build and release engineers, and server administrators. By moving from manual processes to an automated culture, organizations can significantly reduce delivery times (e.g., from weeks to hours). 

#### How to introduce yourself as a DevOps engineer? 
When introducing yourself, clearly state your current experience, your professional background (e.g., previous roles as a system admin or developer), and mention your passion for DevOps.
Emphasize your understanding of the four pillars (automation, quality, monitoring, testing) rather than just listing tools.
